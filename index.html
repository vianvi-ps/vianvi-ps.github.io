  • for 屬性的值必須與表單控制項的 id 屬性值完全相同
  • 優點
    1. 提升使用者體驗：讓使用者不需要精準點擊小小的輸入框，只需要點擊在label 內設定的文字，就能自動跳轉到框框內。讓使用者不需要精準點擊小小的輸入框，特別是在手機等小螢幕裝置上，操作會更方便

CSS基本結構

  • 在HTML結構內設定div的名稱 ex. div class="名稱" 或是 加id ex. h2 id="名稱"
  • 比較
    名字 功能 比喻 性質
    div 區塊容器，用來包住其他內容，方便排版 一個盒子 標籤(Tag)

    class 可重複的名字（可以很多元素用同一個 class） 盒子的衣服款式（很多人可以穿同一款） 屬性(Attributes)
  • 在CSS結構內叫出名稱 ex. .名稱{} 或 直接 h1-6等等(不須.){}
  • 內部每一個的最後面需要+ ;
  • /* 補充說明 */

  4. CSS內建的顏色

% 使用方式+介紹

  1. 定義：是一種相對單位，是某個東西的比例
  2. 優點：可以控制元素的間距，並在不同寬度的螢幕上保持一致的比例


  3. 常見應用 計算基準 注意
    元素尺寸 (width, height) 相對於父容器的尺寸 父容器必須有明確的尺寸(寬度或高度)
    內外邊距 (padding, margin) 水平或垂直，都是相對於父容器的寬度來計算的
    圓角 (border-radius) 相對於元素本身的尺寸 對一個正方形設定 圓角為50% 或 圓角為100% 時，都會形成一個完美的圓形


  4. 特殊情況：不需要父容器尺寸的應用
    • Viewport 單位

      vw (Viewport Width) 和 vh (Viewport Height) 是相對於瀏覽器視窗的尺寸

    • 已定位元素 (position: absolute)

      相對於最近的、已定位的父元素
      (可透過 top: 0; right: 0; bottom: 0; left: 0; 來讓一個絕對定位的子元素填滿父容器)

px (pixel) 使用方式+介紹

  1. 這個單位在 CSS 中就是指一個抽象的「像素」單位
  2. 可以用於設定幾乎所有的 CSS 屬性
  3. 優點
    • 直觀且精確：px 的大小是固定的
    • 易於計算
  4. 缺點
    • 響應性差

@media 使用方式+介紹

  • 定義：針對不同的裝置或條件，套用不同的樣式
  • 格式：媒體類型 and (條件) {
    /* 放想應用的 CSS 樣式 (加 ; )*/
    }
    1. 媒體類型(Media Type)
      all 所有裝置(預設值)
      screen 電腦螢幕、平板和手機等
      print 印表機
      speech 語音合成器(ex. Google 助理)
    2. 條件(Media Feature)
      min-width(height): 螢幕寬度(高度)大於或等於指定值時
      max-width(height): 螢幕寬度(高度)小於或等於指定值時
      orientation: 裝置方向
      • portrait(直向)
      • landscape(橫向)
      prefers-color-scheme: 使用者偏好的顏色模式
      • light
      • dark
      注意敘述的條件

漸變圖片、背景(linear-gradient)

  • 功能：建立漸變背景。將它想成是一種特殊的圖片，這張圖片不是從檔案載入的，而是由瀏覽器根據你的設定動態生成的，可以從一種顏色平滑地過渡到另一種顏色
  • 一般格式：(方向, 顏色1, 顏色2)
  • 加在background後
  • 關鍵字 角度
    向上 to top 0deg
    向下 to bottom 180deg
    向右 to right 90deg
    向左 to left -90deg
  • 半透明圖層，兩個顏色就ok
  • 一個顏色後加 % 漸變；全部顏色後加 % 硬邊(共100 %)
    • 硬邊意思：在漸變中，顏色變化不是平滑過渡的，而是突然從一種顏色跳變到另一種顏色，中間沒有模糊或漸變的過程。
    • 範例：linear-gradient(方向, 顏色A 25%, 顏色B 25%, 顏色B 75%, 顏色A 75%)；結果：0% 到 25% 都是顏色A，25% 到 75% 都是顏色B，75% 到 100% 都是顏色A
  • p.s.顏色可以很多個

"陰影"介紹



水平偏移量（必填)

→陰影相對於元素水平方向的距離，正值往右，負值往左

垂直偏移量（必填）

→陰影相對於元素垂直方向的距離，正值往下，負值往上

模糊半徑（選填）

→陰影的模糊程度，數字越大陰影越柔和擴散。預設為0（沒有模糊）

擴散半徑（選填）

→陰影大小的擴散範圍，正值會讓陰影變大，負值會縮小陰影

顏色（選填，但建議加）



內陰影關鍵字 inset（選填）

→如果加上 inset，陰影會變成元素內部的陰影，而非外部

佈局模式-1(flex)

  • 核心概念是父容器(Container)和子項目(Item)
    1. 父容器：容器會自動地將它的所有子元素變成彈性項目，並將它們預設排成一條直線(水平方向)
    2. 子項目：彈性項目會根據父容器的設定自動調整它們的尺寸和位置
  • justify-content搭配
    功能：控制子項目在主軸（預設為水平方向）上的對齊和間距
    屬性值 功能
    flex-start 所有項目靠左對齊
    flex-end 所有項目靠右對齊
    center 所有項目置中對齊
    space-between 項目之間平均分配空間，兩端沒有間距
    space-around 項目之間和兩端都有平均的間距
  • align-items搭配
    功能：控制子項目在交叉軸（預設為垂直方向）上的對齊
    屬性值 功能
    flex-start 所有項目靠上對齊
    flex-end 所有項目靠下對齊
    center 所有項目垂直置中
  • flex-direction搭配
    功能：改變子項目的排列方向，讓主軸變成垂直或水平
    屬性值 功能
    row 預設值，項目水平排列
    column 項目垂直排列

偽類-1(child & type)

  1. 考慮的是父元素下的所有子元素
    • :first-child
      • 選取父元素下的第一個子元素
    • :last-child
      • 選取父元素下的最後一個子元素
    • :nth-child(n)
      • 選取第 n 個子元素，n 可以是數字、關鍵字或數學公式
    • :nth-last-child(n)
      • 從最後一個元素開始往前算

  2. 只會計算相同標籤類型的兄弟元素
    • :first-of-type
      • 父元素下相同類型的第一個子元素
    • :last-of-type
      • 選取其父元素下相同類型的最後一個子元素
    • :nth-of-type(n)
      • 選取其父元素下相同類型的第 n 個子元素

屬性值 功能 專門用於 a 連結標籤

注意！！！

為了保護網路安全，必須控制 :link 和 :visted 內的屬性值。只能修改顏色相關，例如

  • color
  • background-color
  • border-color

原因：過去，惡意網站可以透過 CSS 來檢查是否訪問過某些網頁。他們會設定 a:visited 樣式，然後用 JavaScript 檢查連結是否變色。如果變色了，就代表曾經訪問過這個網站，他們就能藉此竊取瀏覽歷史。

transition(過渡)屬性
功能： 控制一個 CSS 屬性從一個狀態平滑地過渡到另一個狀態
格式： [transition-property] [transition-duration] [transition-timing-function] [transition-delay]
意義： 哪個 CSS 屬性要進行過渡 過渡持續的時間 過渡的速度曲線 過渡開始前的延遲時間
搭配： background-color、all... 秒 (s) 或毫秒 (ms)
  1. ease：預設值，開始和結束都比較慢
  2. linear：速度保持一致
  3. ease-in：開始時比較慢
  4. ease-out：結束時比較慢
  5. ease-in-out：從緩慢啟動、加速到中間最快、再到緩慢結束
transform 搭配
功能：對元素進行形狀上的變換，但不會影響到頁面布局
備註：可以放一起，但順序決定發生時間

position搭配

  1. static (靜態定位)(所有元素的預設值)
    • 按照 HTML 程式碼的順序，由上而下、由左而右地顯示
    • top, bottom, left, right 這類定位屬性對它無效
  2. relative (相對定位)
    • 元素移動後，它原本在頁面上的空間仍然會被保留，這表示它移動後，周圍的元素不會填補它原本留下的空位
  3. absolute (絕對定位)
    • 會將元素從正常的文檔流中完全移除，這表示它不再佔據任何空間，周圍的元素會像它不存在般重新排列
    • 會根據它最近的、已定位的祖先元素來定位
    • 已定位的祖先元素指的是 position 不為 static 的父、祖父等元素
    • 如果找不到任何已定位的祖先元素，元素就會相對於 body 進行定位
  4. fixed (固定定位)
    • 會將元素從文檔流中移除，就像 absolute
    • 元素會永遠相對於瀏覽器視窗進行定位，即使滾動頁面，元素也會固定在同一個位置不動
  5. sticky (黏性定位)
    • 元素只在特定條件下才會「黏住」，一開始是跟著流動的，只有碰到設定好的「黏著點」時，才會固定不動。
    • 必須同時設定一個黏著點
